Pine is a static site generator. It uses Branches at its core to convert a directory of content to a static collection of HTML and CSS. You can download the latest .phar executable here.

After that, you can add the executable to the $PATH and just use pine from the command line. At this point, you can create a new site with:

pine new blog # Where blog is the name of your site
cd blog

To serve the file at http://127.0.0.1:8080 for development, use the serve command:

pine serve

Finally, use the command build to generate the static site.

pine build

Configuration

To generate a website, Pine needs content and a theme. A pine website will look like this:

The configuration file allows you to customize the name of the directories. The default configuration looks like this:

theme: default
content: content
build: build
plugins: []

The options content and build refer to what directories are to be used for storing respectively the content files and the generated site. With theme, you can choose the active theme, which is to be found in the themes directory. And finally, plugins is a list of PHP classes that pine will use as plugins.

Content

Content files reside in the content directory (or in the directory that is set in the configuration file under content). As described in the documentation of Branches, every folder contained in this directory is a post, where the folder itself is considered the root post.

Pine will look for a markdown file in every post, to be used as the content file. This file will contain a YAML front matter where the post meta data can be specified, and a markdown body.

---
title: A post title
layout: blog
---

The post *content*.

As you can see here, a title is included, as well as a layout. Pine will use this layout, or it will use default if none is specified. More on layouts can be found on the next section.

Every other file in a post is considered an attachment.

Themes

A single pine theme will contain some configuration, public files (like JavaScript, or CSS), and twig templates.

When the site is generated, all contents of the theme public directory will be moved to the generated site directory, where they will be under a folder called theme.

The templates can access the configuration of the theme and the site by using the variables theme and site, respectively. For example, a theme could support both "serif" and "sans-serif" styles by having the configuration on the theme config directory:

style: serif

And in the templates, the configuration value can be accessed with:

<body class="style-{{ theme.config.style }}">
    <h1>{{ site.config.title }}</h1>
    ...
